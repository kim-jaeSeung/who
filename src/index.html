⚠️ 문제 해결

- 외부 API 로딩 속도, 실시간 주가 변동 기능에 대한 성능 문제

외부 API를 통해 주식 가격을 받아와 프론트엔드에서 이틀치 데이터를 비교하며 실시간 주가 변동 기능을 구현하려 했으나, API 캐싱 작업을 해도 느린 로딩 속도와 과도한 메모리 사용량 문제가 발생했습니다. 이를 해결하기 위해, 외부 API 호출 및 실시간 주가 변동 로직을 백엔드로 이전하고, 처리된 데이터를 데이터베이스에 저장하는 방식으로 개선하고 스피너를 추가해 사용자 경험을 향상 시켰습니다.

👍 결과

- 로딩 속도 개선

외부 api를 기반으로 만든 데이터를 데이터 베이스에 담아둬서 프론트엔드에서 직접 API를 호출하고 데이터를 처리하는 부담이 줄어들어 로딩 속도가 크게 단축되었습니다.

- 클라이언트 성능 향상

프론트엔드에서의 데이터 처리 및 저장 부담이 백엔드로 이전되어 클라이언트 측의 성능을 효율적으로 향상시켰습니다.

- 사용자 경험 향상

첫 방문 사용자도 지연 없이 서비스를 이용할 수 있게 되어 전반적인 사용자 경험이 향상되었습니다.

개인 포트폴리오 웹사이트를 JavaScript와 TypeScript를 기반으로 직접 구현하였습니다. 정적인 정보만 나열하는 방식이 아닌, 주요 콘텐츠(경력 사항, 교육 이수 내역 등)를 JSON 형식의 구조화된 데이터로 관리하고, 이를 클라이언트 측에서 API 호출을 통해 동적으로 렌더링하는 구조로 설계했습니다. 이렇게 함으로써 콘텐츠 변경 시 코드 수정 없이도 데이터만 업데이트하면 사이트가 최신 상태로 유지되도록 하였습니다. GitHub Actions를 통해 CI 환경을 구축하였습니다. 테스트 측면에서는 Jest를 도입하여 API 호출 모듈의 테스트를 진행하여 데이터 렌더링 안정성을 확보하였습니다.

👍 프로젝트 성과 및 개선 사항

- API 기반의 데이터 구조화 및 동적 렌더링 설계 - TypeScript 기반의 안정적인 코드 작성 - GitHub Actions 기반 자동화된 CI 구축 - Jest 테스트를 통한 API 로직의 안정성 확보 - 추후에는 모든 정적인 정보들을 json 파일로 관리하여, 동적 렌더링을 구현할 예정입니다.
